The project documentation is designed with a **Simplicity First** and **Goal-Driven** approach, primarily targeting AI Agents and new developers. 

It consolidates all project knowledge into four core files to avoid fragmentation.

## Core Rules for Maintenance
- **Surgical Changes**: Update [[architecture-overview]] if Database Schema changes.
- **Prompt Integrity**: Update [[ai-integration-details]] if prompt logic in `application/services/` changes.
- **Minimalism**: Do not create new files; maintain the existing 4-file structure in `raw/`.
